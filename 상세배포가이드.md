# 🚀 GitHub Pages 배포 가이드 (모바일로 바로 보기)

## 📱 최종 결과
이 가이드를 따라하시면 5-10분 안에 모바일에서 QR코드로 접속 가능한 URL이 생성됩니다!

예시: `https://사용자명.github.io/hospital-guide/`

---

## 🎯 1단계: GitHub 계정 만들기

### 이미 계정이 있으시면 이 단계는 건너뛰세요!

1. 브라우저에서 **https://github.com** 접속
2. 우측 상단 **"Sign up"** 버튼 클릭
3. 이메일 주소 입력
4. 비밀번호 생성
5. 사용자명 입력 (예: `bonafide-clinic`)
6. 이메일 인증 완료

---

## 🗂️ 2단계: 새 저장소(Repository) 만들기

1. GitHub에 로그인 후 우측 상단 **"+" 버튼** 클릭
2. **"New repository"** 선택

![스크린샷: New repository 버튼 위치]

3. Repository 설정:
   ```
   Repository name: hospital-guide
   Description: 병원 안내문 웹사이트
   ✅ Public (반드시 Public으로 선택!)
   ❌ Add a README file (체크 안 함)
   ```

4. **"Create repository"** 버튼 클릭

![스크린샷: Repository 생성 화면]

---

## 📤 3단계: 파일 업로드하기

Repository가 생성되면 빈 페이지가 나타납니다.

### 방법 1: 웹에서 직접 업로드 (추천!)

1. **"uploading an existing file"** 링크 클릭

![스크린샷: uploading an existing file 링크]

2. 다운로드 받은 **hospital-guide 폴더**를 열기

3. **폴더 안의 모든 파일**을 선택하여 드래그 앤 드롭
   - index.html
   - body-parts.html
   - injection-precautions.html
   - non-covered.html
   - bone-injection.html
   - README.md
   - 빠른시작가이드.md
   - 모든 .png 이미지 파일들 (14개)

4. 잠시 기다리면 파일 목록이 나타남

5. 페이지 하단으로 스크롤
   - Commit message: "병원 안내문 웹사이트 업로드"
   - **"Commit changes"** 버튼 클릭

![스크린샷: 파일 업로드 화면]

### ✅ 확인: 파일이 제대로 업로드 되었나요?
Repository 메인 화면에서 파일 목록을 확인하세요.

---

## ⚙️ 4단계: GitHub Pages 활성화하기

이제 웹사이트를 인터넷에 배포합니다!

1. Repository 상단의 **"Settings"** 탭 클릭

![스크린샷: Settings 탭 위치]

2. 왼쪽 메뉴에서 **"Pages"** 클릭 (아래쪽에 있음)

![스크린샷: Pages 메뉴 위치]

3. "Build and deployment" 섹션에서:
   ```
   Source: Deploy from a branch
   Branch: main (선택)
   Folder: / (root) (선택)
   ```

4. **"Save"** 버튼 클릭

![스크린샷: GitHub Pages 설정 화면]

5. 페이지 새로고침 (F5)

6. 상단에 초록색 박스가 나타나면 성공!
   ```
   ✅ Your site is live at https://사용자명.github.io/hospital-guide/
   ```

![스크린샷: 배포 완료 화면]

## 🎉 5단계: 모바일로 확인하기

### 방법 1: URL 직접 입력
1. 스마트폰 브라우저 열기
2. URL 입력: `https://사용자명.github.io/hospital-guide/`
3. 즐겨찾기에 추가!

### 방법 2: QR 코드 생성 (추천!)

#### 네이버 QR코드 (무료, 간편)
1. 네이버에서 **"QR코드 생성"** 검색
2. 첫 번째 결과 클릭
3. URL 입력란에 GitHub Pages URL 붙여넣기
4. **"QR코드 만들기"** 클릭
5. **"다운로드"** 버튼 클릭

#### 다른 QR 코드 생성 사이트
- https://www.qr-code-generator.com/
- https://www.the-qrcode-generator.com/

## 🔧 7단계: 나중에 수정하는 방법

### 이미지 파일 교체
1. GitHub Repository 접속
2. 교체할 이미지 파일 클릭
3. 우측 상단 **휴지통 아이콘** 클릭 → 삭제
4. Repository 메인으로 돌아가기
5. **"Add file" → "Upload files"** 클릭
6. 새 이미지 업로드 (같은 파일명으로!)
7. "Commit changes" 클릭

⏰ **1-2분 후 자동으로 업데이트됩니다!**

### HTML 내용 수정
1. 수정할 HTML 파일 클릭 (예: index.html)
2. 우측 상단 **연필 아이콘** (Edit) 클릭
3. 내용 수정
4. 하단 "Commit changes" 클릭

## 🆘 문제 해결

### ❌ "404 - Page not found" 에러
**원인:** GitHub Pages가 아직 배포 중
**해결:** 2-3분 기다린 후 다시 접속

### ❌ 이미지가 안 보임
**원인:** 파일명이 잘못되었거나 업로드 안 됨
**해결:** 
1. Repository에서 파일 목록 확인
2. 모든 .png 파일이 있는지 확인
3. 없으면 다시 업로드

### ❌ URL을 모르겠어요
**확인 방법:**
1. Settings → Pages로 이동
2. 상단에 URL 표시됨
3. 형식: `https://사용자명.github.io/hospital-guide/`

### ❌ Public으로 만들었는데 안 보여요
**확인:**
1. Settings → General
2. "Danger Zone" 섹션에서 "Change visibility" 확인
3. Public인지 확인

## 📊 추가 기능 (선택사항)

### URL 단축하기
긴 GitHub URL을 짧게 만들기:

1. **Naver 단축URL**
   - https://me2.do 접속
   - GitHub Pages URL 입력
   - 짧은 URL 생성 (예: me2.do/abc123)

2. **Bitly**
   - https://bitly.com 접속
   - 무료 계정 생성
   - URL 단축

### Google Analytics 추가
방문자 통계를 보고 싶다면:
1. Google Analytics 계정 생성
2. 추적 코드 받기
3. index.html의 `</head>` 태그 바로 앞에 코드 삽입
